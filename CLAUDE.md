## Coding Guidelines (Karpathy Rules)

### 1. Think Before Coding

- 모르면 추측하지 않는다. 문서 확인 → 없으면 사용자에게 질문.
- 가정을 명시적으로 밝힌다.
- 해석이 여러 개면 조용히 하나 고르지 말고 제시한다.
- 더 단순한 방법이 있으면 말한다. 필요하면 반론한다.

### 2. Simplicity First

- 요청받은 것만 구현한다. 투기적 기능 금지.
- 한 번 쓰는 코드에 추상화 금지.
- 불가능한 시나리오에 대한 에러 처리 금지.
- 200줄로 쓴 걸 50줄로 할 수 있으면 다시 쓴다.

### 3. Surgical Changes

- 관련 없는 코드/주석/포매팅을 "개선"하지 않는다.
- 깨지지 않은 것을 리팩토링하지 않는다.
- 기존 스타일에 맞춘다.
- 내 변경으로 생긴 미사용 코드만 제거한다. 기존 데드코드는 언급만.

### 4. Goal-Driven Execution

- 작업을 검증 가능한 목표로 변환한다.
- 멀티스텝 작업은 `[Step] → verify: [check]` 형식으로 계획을 세운다.
- 성공 기준이 약하면 ("되게 해줘") 명확히 할 때까지 확인한다.
